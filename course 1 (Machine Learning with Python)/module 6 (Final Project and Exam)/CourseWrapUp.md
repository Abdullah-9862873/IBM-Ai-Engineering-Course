## Course Summary

### Introduction to Machine Learning

- **Machine Learning (ML)** is a subset of Artificial Intelligence (AI) that uses data and algorithms to allow computers to imitate how humans learn and make decisions, gradually improving their accuracy.

### Applications of Machine Learning

- **Healthcare**: Doctors use machine learning to prescribe the correct medicine to patients
- **Finance**: Bankers use machine learning to decide whether to approve or reject loan applications
- **E-commerce**: Businesses use machine learning to generate customer recommendations

### Machine Learning Types

- Supervised Learning
- Unsupervised Learning
- Semi-supervised Learning
- Reinforcement Learning

### Machine Learning Tools

- ML tools provide functionalities for ML pipelines
- Include modules for data preprocessing, building, evaluating, optimizing, and implementing models
- Use algorithms to simplify complex tasks like handling big data, statistical analyses, and predictions

---

## Supervised Learning: Regression

- **Regression** models the relationship between a continuous target variable and explanatory features
- **Simple Regression**: Single independent variable estimates a dependent variable (linear or nonlinear)
- **Multiple Regression**: Two or more independent variables estimate a dependent variable

---

## Supervised Learning: Classification

- **Classification** is a supervised ML method that predicts categorical labels on new data
- Labels form a categorical variable with discrete values
- Applications: Email filtering, speech-to-text, handwriting recognition, biometric identification, document classification

### Classification Algorithms

#### K-Nearest Neighbors (KNN)
- Supervised ML algorithm that labels data points based on similarity to neighbors
- Used for both classification and regression  

#### Support Vector Machines (SVM)
- Supervised learning for classification and regression
- Maps data instances as points in multidimensional space
- Good for speech recognition, anomaly detection, noise filtering

#### Decision Trees 
- Each internal node corresponds to a test
- Each branch corresponds to the test result
- Each leaf node assigns data to a class 
 
#### Regression Trees
- Predicts continuous values rather than discrete classes
- Built by considering features of a dataset one by one
- Created by recursively splitting dataset to maximize information gain
- Minimizes randomness of classes assigned to split nodes

---

## Unsupervised Learning: Clustering

- **Clustering** automatically groups data points into clusters based on similarities
- Applications: Identifying music genres, segmenting user groups, analyzing market segments
- Complementary to classification and regression

### Clustering Methods

- **Partition-based**: K-means clustering
- **Density-based**: DBSCAN
- **Hierarchical**: Agglomerative and Divisive

---

## Dimension Reduction

- Simplifies visualization of high-dimensional clustering
- Aids feature engineering and improves model quality
- Reduces number of features required for a data model
- Techniques: PCA, t-SNE, UMAP

---

## Model Evaluation

### Classification Metrics
- Accuracy
- Confusion Matrix
- Precision
- Recall
- F1 Score

### Regression Metrics
- MAE (Mean Absolute Error)
- MSE (Mean Squared Error)
- RMSE (Root Mean Squared Error)
- R-squared

---

## Model Validation

### Key Concepts
- **Model Validation**: Optimizes model without jeopardizing ability to predict on unseen data
- **Data Snooping**: Checking test data performance before optimization is complete (form of data leakage)
- **Proper Validation**: Train on training data, only test on unseen test data after satisfied with training

### Cross-Validation
- K-fold cross-validation
- Stratified cross-validation
- Prevents overfitting

### Regularization
- Ridge Regression (L2)
- Lasso Regression (L1)
- Prevents overfitting by adding penalty terms

---

## Key Takeaways

1. **Problem Selection**: Choosing the right ML technique depends on problem type, data, resources, and desired outcome

2. **Classification vs Regression**: The distinguishing feature is the characteristic of the target variable (discrete vs continuous)

3. **Model Quality**: Evaluate using appropriate metrics for classification (accuracy, precision, recall, F1) and regression (MAE, MSE, RMSE, R²)

4. **Validation**: Always validate models properly to avoid data leakage and overfitting

5. **Complementary Techniques**: Clustering, dimension reduction, and feature engineering work together to improve model performance
